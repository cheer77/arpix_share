Pixel Perfect Strategy

Pixel-perfect layout means matching the design while maintaining correct web layout behavior.

Rules:

* Respect Figma spacing values
* Respect typography sizes
* Respect layout alignment
* Use gap instead of stacked margins
* Avoid fixed heights that break responsiveness

Priority order:

1. Correct layout structure
2. Correct spacing
3. Correct typography
4. Responsive stability
5. Pixel-perfect alignment

Never sacrifice layout flexibility just to match fixed Figma height values.
--------------------------------------------------

Responsive Rules
* Use mobile-first approach
* Breakpoints: 576px, 768px, 1024px, 1280px
* Use Flexbox and CSS Grid

--------------------------------------------------

Fluid Layout Rules
* Do NOT blindly copy fixed width or height from Figma
* Prefer max-width instead of fixed width
* Prefer min-height instead of fixed height
* Let content define height whenever possible
* Use Flexbox or Grid for flexible layouts
Example:

Container Enforcement Rule

Every top-level layout section MUST contain a `.container`.

Correct structure:

<section class="hero">
  <div class="container">
    ...
  </div>
</section>

Rules:

* Never place content directly inside `<section>` without a container
* Containers maintain layout consistency
* The `.container` class must always be used instead of creating new wrapper widths
* Never duplicate container logic

If SCSS container mixins exist in the project, prefer:

@include container;

SCSS AC() Function Usage Rule
* Use the ac() function for fluid typography, spacing, and sizing
* Parameters: $startSize, $minSize, $minBreakPoint, $startBreakPoint, $flag
* Always use ac() for elements that must scale between breakpoints
* Combine with SCSS variables for maintainability ($maxWidthContainer, $phoneWidthFontSize)
* Example: font-size: ac(24px, 16px);
* Use $flag: true for container-based scaling: font-size: ac(24, 16, $phoneWidth, $maxWidth, true);
* Never hardcode scaling for elements that should be fluid
* Combine with media queries only if overriding default breakpoints

This function MUST be used for:

* font-size
* spacing
* margins
* padding
* widths when scaling is required
* heights ONLY when truly necessary

Rules:

* Never hardcode scalable values
* Prefer ac() instead of fixed px values
* Respect project variables like:
  $maxWidth
  $phoneWidth
  $maxWidthContainer
  $phoneWidthFontSize

Correct example:

font-size: ac(32px, 20px);
padding: ac(80px, 40px) 0;
gap: ac(40px, 20px);

Incorrect example:

font-size: 32px;
padding: 80px;
