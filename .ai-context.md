Pixel Perfect & Figma Protocol (via MCP)

Strict Alignment

Unit Conversion

--------------------------------------------------

Responsive Rules
* Use mobile-first approach
* Breakpoints: 576px, 768px, 1024px, 1280px
* Use Flexbox and CSS Grid
* Avoid unnecessary position: absolute

--------------------------------------------------

Fluid Layout Rules
* Do NOT blindly copy fixed width or height from Figma
* Prefer max-width instead of fixed width
* Prefer min-height instead of fixed height
* Let content define height whenever possible
* Use Flexbox or Grid for flexible layouts
Example:
max-width: 1280px; width: 100%;
min-height: 100vh;

--------------------------------------------------

Container System Rules
* Use a consistent container system
* Wrap content in .container
Example:
<section class="hero">
  <div class="container">
    ...
  </div>
</section>

.container {
  width: 100%;
  max-width: 1280px;
  margin: 0 auto;
  padding: 0 16px;
}

--------------------------------------------------

Figma Auto-Layout Interpretation
* Horizontal → display: flex; flex-direction: row
* Vertical → display: flex; flex-direction: column
* Use gap instead of multiple margins
Example:
display: flex; gap: 24px;

--------------------------------------------------

Spacing System Rules
* Prefer gap for spacing in flex/grid
* Use padding for internal spacing
* Use margin for spacing between independent components
* Avoid stacking multiple margins

--------------------------------------------------

SCSS Mixins Usage
* Always detect existing mixins
* Prefer @include instead of repeating logic
* Do not duplicate responsive logic if mixins exist
* Prefer project utilities, mixins and variables over writing new CSS

--------------------------------------------------

Typography System Rules
* Follow typography values from Figma
* Do not generate random font sizes
* Use consistent scale for headings and text
* Use variables for font sizes when available

--------------------------------------------------

Image Handling Rules
* Use <img> for content images
* Use CSS background images for decorative elements
* Use inline SVG when icons require styling or animation
* Do not embed large SVGs inside <img> if styling is needed
* Always include alt attributes

--------------------------------------------------

Accessibility Rules
* Always use alt attributes
* Use aria-label when necessary
* Ensure buttons and links are accessible
* Maintain semantic HTML structure

--------------------------------------------------

SCSS AC() Function Usage Rule
* Use the ac() function for fluid typography, spacing, and sizing
* Parameters: $startSize, $minSize, $minBreakPoint, $startBreakPoint, $flag
* Always use ac() for elements that must scale between breakpoints
* Combine with SCSS variables for maintainability ($maxWidthContainer, $phoneWidthFontSize)
* Example: font-size: ac(24, 16);
* Use $flag: true for container-based scaling: font-size: ac(24, 16, $phoneWidth, $maxWidth, true);
* Never hardcode scaling for elements that should be fluid
* Combine with media queries only if overriding default breakpoints
